▶ MIME(Multipurpose Internet Mail Extensions) 이란?
    ==> Multipurpose Internet Mail Extensions의 약자이며, 간단히 말해 MIME 란 파일 변환을 뜻하는 것이다.
           MIME 라는 용어가 사용되기 시작한 이유는 e메일과 함께 첨부할 파일을 텍스트 문자로 변환해서 e메일 시스템을 통해 전달하기 위해 개발되었기 때문에 Internet Mail Extensions(확장) 이라는 용어를 쓰게 되었다. 그러나 현재는 기술이 더욱더 발전하여 웹을 통해 여러 다양한 형태의 파일을 전달하는데 사용하게  되었지만 용어는 그대로 MIME(Multipurpose Internet Mail Extensions) 라고 사용하고 있는 것이다.


▶ MIME 사용 이유
   ==> 초창기에 나온 Internet Mail 에 있어서 첨부파일 형태는 텍스트 파일만 사용하였으므로, 텍스트 파일을 주고 받는데 있어서 ASCII로 공통된 표준에 따르기만 하면 문제가 없었으나, 시대가 흐름에 따라 네트워크를 통해 ASCII가 아닌 다양한 형태의 바이너리 파일을 보내는 경우가 생겨나게 되었다. 음악파일, 동영상파일, 영화파일, 워드파일 등등등은 ASCII만을 사용해서는 파일 전송이 안되기 때문에, 텍스트 기반으로 사용하던 기존의 시스템에서도 음악파일, 동영상파일, 영화파일, 워드파일 등등등을 문제 없이 전달하기 위해서는 음악파일, 동영상파일, 영화파일, 워드파일 등등등을 텍스트로의 변환이 필요하게 되었다. 그래서 용어는 예전 그대로 MIME(Multipurpose Internet Mail Extensions) 로 사용하지만 향상된 MIME 기술을 사용하여 파일변환을 해주는 것이 등장하게 된 것이다.  


▶ 인코딩(Encoding) 과 디코딩(Decoding)
   인코딩(Encoding) 이란 사람이 인지할 수 있는 문자(언어)를 약속된 규칙에 따라 컴퓨터가 이해하는 언어(0 과 1)로 이루어진 코드로 바꾸는 것을 통틀어 말한다. 
   즉, 인코딩(encoding)은 정보의 형태나 형식을 표준화, 보안, 처리 속도 향상, 저장 공간 절약 등을 위해서 다른 형태나 형식으로 변환하는 처리 혹은 그 처리 방식을 말한다.
 
   디코딩(Decoding) 은  인코딩(Encoding)의 반대되는 개념으로 복호화, 역코드화의 의미를 가지고 있다.   
   바이너리 파일을 텍스트 파일로 변환하는 것을 디코딩(Decoding) 이라 하고, 
   텍스트 파일을 바이너리 파일로 변환하는 것을 인코딩(Encoding) 이라고 한다.


▶ Content-type
    MIME 를 사용하여 인코딩(Encoding)한 파일은 파일의 앞부분(header)에 Content-type 정보를 담게 되어 있는데 Content-type 은 여러가지 타입이 있다.

웹 브라우저에서 서버에 접속하여 html 문서를 요청하면서 html문서에 있는 이미지 파일의 경로를 불러올 수 있다. 이러한 과정에서 이미지의 경로에 있는 파일이 웹브라우저에서 지원되는 MIME-Type이라면 웹브라우저를 이용하여 열어볼 수 있다. 바이너리파일(음악 파일, 무비 파일, 워드 파일 등) 또한 마찬가지 이다. 주로 쓰고 있는 대부분의 포맷인 .gif .jpg .mov 등등의 파일들은 웹 브라우저에서 무리없이 열리게 되는데 브라우저에서 지원하지 못하는 유형은 따로 지정해줘야 한다.
